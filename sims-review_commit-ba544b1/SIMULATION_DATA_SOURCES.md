## Real-World Data Files

Three CSV files in the repo root supply real market data. Only the year-long studies consume them; short-duration simulations use purely synthetic price/rate models.


| File                     | Source         | Content                                         | Resolution             | Coverage                | Rows  |
| ------------------------ | -------------- | ----------------------------------------------- | ---------------------- | ----------------------- | ----- |
| `btc-usd-max.csv`        | CoinGecko      | BTC/USD price, market cap, volume               | Daily                  | 2013-04-28 ‚Üí 2025-09-25 | 4,533 |
| `rates_compute.csv`      | Dune Analytics | AAVE V2 USDC variable borrow rate + supply rate | Daily (non-contiguous) | 2021-01 ‚Üí 2023-10       | 1,783 |
| `dune_query_6227486.csv` | Dune Analytics | BTC/USD price                                   | 1-minute               | 2025-10-10 (single day) | 1,441 |


`**optimal_range_lookup_corrected.csv`** ‚Äî pre-computed Uniswap V3 LP range bounds, referenced by `pool_rebalancer.py` but **not present** in the repo. Missing file causes `FileNotFoundError` at runtime if the ALM rebalancer's optimal-range code path is hit.

`**dune_query_6227486.csv`** ‚Äî not loaded by any simulation script. Likely used for one-off analysis or intended for future use.

## Data Usage by Simulation

### Year-Long Studies (Studies 1‚Äì10): `sim_tests/full_year_sim.py` + `run_study_*.py`

**BTC prices** ‚Äî `btc-usd-max.csv` filtered by year ‚Üí one price/day ‚Üí linearly interpolated to per-minute via `get_btc_price_at_minute()`. Fallback if CSV missing: synthetic linear path.


| Year | BTC range                   | Duration |
| ---- | --------------------------- | -------- |
| 2021 | $29,002 ‚Üí $46,306 (+59.6%)  | 365 days |
| 2022 | $46,320 ‚Üí $16,604 (‚àí64.2%)  | 365 days |
| 2024 | $42,208 ‚Üí $92,627 (+119%)   | 365 days |
| 2025 | $93,508 ‚Üí $113,321 (+21.2%) | 268 days |


**AAVE rates** ‚Äî `rates_compute.csv` filtered by year, USDC symbol ‚Üí daily variable borrow rate ‚Üí linearly interpolated to fill gaps ‚Üí applied per-minute via `get_historical_rate_at_minute()`. Fallback: flat 5% APR.

- *Symmetric studies (1‚Äì5)*: both HT and AAVE agents use the same historical AAVE rate.
- *Asymmetric studies (6‚Äì10)*: HT uses Advanced MOET dynamic rate; AAVE uses historical AAVE rate from CSV.

### `balanced_scenario_monte_carlo.py` (Primer ¬ß4.2, Figure 2)

**No real-world data.** BTC price generated by `BTCPriceDeclineManager`: linear decline from $100k to target ($76,342.50) over 60 min with random per-minute perturbations (`decline_rates = [-0.0054, ..., -0.0040]`, 5% chance of max decline ‚àí0.95%/min, ¬±0.05% noise). Final 20% of duration: price steered toward target. AAVE borrow rate: constant from engine defaults (protocol's `calculate_borrow_rate()`), not CSV-driven.

### `flash_crash_simulation.py` (Flash Crash Stress Test)

**No real-world data.** All price dynamics are hardcoded:

- *BTC*: $100k ‚Üí linear drop over 5 min to floor (scenario-dependent: ‚àí12/20/25%) ‚Üí hold at floor for 20 min ‚Üí exponential recovery `1‚àí(1‚àíp)^1.5` with ¬±2% uniform noise, clamped to [floor, $100k].
- *YT oracle*: gradual decline to `yt_floor_price`, ¬± `oracle_volatility` noise, ~12%/min wick probability, exponential recovery `1‚àí(1‚àíp)^2`.
- *Liquidity*: predetermined throttling schedule (30% ‚Üí 60/70/80% peak ‚Üí capped 95%), piecewise-linear recovery.

### `hourly_test_with_rebalancer.py` (Pool Rebalancer 36h Test)

**No real-world data.** Uses engine's `BTCPriceDeclineManager` (same synthetic model as Monte Carlo sim). Duration: 36 hours.

### `aave_leverage_strategy_sim_v2.py`

**Yes ‚Äî BTC prices only.** Loads `btc-usd-max.csv` for 2024 data (`_load_2024_btc_data`). AAVE rates: not loaded from CSV; uses engine defaults.

### Always synthetic (all simulations)

- YT pricing: `V(t) = V‚ÇÄ √ó (1 + 0.10)^(t/525600)` ‚Äî 10% APR rebasing formula
- Uniswap V3 pool state: initialized from config params, not historical LP data
- MOET pricing: formula-based (`k √ó geometric_mean(backing_assets)`) or hardcoded $1
- Agent behavior: parameterized HF thresholds, not learned/calibrated from real user data
- Liquidation mechanics: simplified model (50% debt, 5% bonus), not real AAVE protocol
